2.게임 디자인 및 기획
-게임 플레이
(스테이지 구성 단순도를 보여주며)
4개의 스테이지 로 구성 각 스테이지 이후 새로운 무기를 획득
마지막 보스전 이후 클리어

-게임 스토리
주인공이 연구소를 탈출한다는 내용

-레벨 디자인
(맵 사진 보여주며) 
각 스테이지 설명
1스테이지 가장 약한 적을 배치
2스테이지 점프맵을 통해 플레이어 이동조작능력을 요구
3스테이지 다수에 적을 배치하여 플레이어의 회피능력과 슈팅능력을 요구
4스테이지 강력한 보스를 배치하여 전체 스테이지에 요구능력을 모두 발휘

-게임 아트 스타일
(플레이어 모델링, 적 모델링, 무기 모델링 사진)
(맵 텍스쳐 사진)

최대한 리얼한 텍스쳐 사용
3D 모델링은 에셋 스토어를 통해 구함

-UI 디자인 소개
심플하고 깔끔한 UI를 추구
(메인화면, 설정창, 랭킹창, 클리어 화면)
각 화면 구성 설명

메인화면엔 
인게임으로 들어가는 시작버튼
설정창으로 가는 설정버튼
랭킹창으로 가는 랭킹버튼
종료버튼으로 구성

설정창 
각 소리별로 오디오 믹서를 활용하여 소리를 제어 
플레이어의 마우스 감도 조절 기능도 있습니다.
랭킹창 기능 및 UI 설명

SQLite를 통해 DB관리된 내용을 랭킹순으로 출력해줍니다
클리어 시간을 기준으로 등수가 정해지고
등수, 클리어 시간, 클리어 날짜 별로 오름차순 내림차순 정렬기능과
아이디 검색 기능도 있습니다.

(인게임 화면 UI 사진 보여주며)
인게임 화면구성 설명
체력바, 
오른쪽 하단에 체력바와 쉴드바가 있으며
무기 인벤 작동방식
각 무기 변경시 이벤토리가 바뀌며 작동합니다.

(플로우 차트 보여주며)
서로 넘어갈수 있도록 제작하였다.

3.기술적 요소
-기술 스택
사용 프로그래밍 언어 C#
게임 엔진 유니티 
사용한 이유 팀원들이 만져본 경험이 있어서 사용함

-프로그래밍 및 시스템 아키텍처
(코드 전체 구성 그림 보여주며)
플레이어 + 적 AI코드 + 체력(데미지) 시스템 + 아이템 시스템 + 스테이지 컨트롤 + 사운드 매니저)
로 구성되어 있고 
플레이어와 적 간에는 데미지 시스템을 통한 상호작용 
데미지 시스템을 통한 적의 사망유무를 체크해 스테이지 컨트롤이 전체 게임 진행을 한다
게임속 플레이어와 적, 맵의 상호작용속 생기는 사운드는 사운드 매니저를 통해 관리된다.

각 코드를 팀원별로 역활을 맡아 제작함

5.게임 테스트 및 피드백

-테스트 과정
매 회의 마다 내부적 버그해결 및 피드백을 통해 문제 해결

-내부 피드백
게임의 재미요소에 대한 피드백으로
기존 스테이지안 아이템 배치에서 몬스터 처치시 아이템 획득으로 변경

-버그 및 문제 해결
카메라 벽 뚫는 문제
마우스 움직임을 통한 카메라 버그는 코드적으로 해결
플레이어 이동에 따른 카메라 관통 버그는 특정 지점에서만 발생하여
맵에 가드라인을 생성하여 벽을 관통하여도 맵밖을 볼수 없도록 해결
