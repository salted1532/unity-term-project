프로젝트 목표 - 
3D 슈팅 어드벤처 게임의 3인칭 버전
기획에 맞는 게임을 "완성" 하는것이 목표
게임의 완성도를 최대한 올릴것

프로젝트 개요 및 특징 - 
10가지 이하의 버튼으로만 으로 플레이 가능한 간단한 조작
다양한 움직임을 통한 속도감 있는 플레이
다양한 무기를 통한 전략적인 전투 


2. 프로젝트 목표
-속도감있는 플레이를 위한 다양한 움직임이 가능한 플레이어 제작
1. 3인칭 카메라 움직임 구현
2. 플레이어 이동 구현
3. 플레이어 점프, 더블 점프 구현
4. 플레이어 대쉬 구현

-4개의 스테이지로 구성한 하나의 맵 제작
1. 적 몬스터 구현
2. 3인칭 시점에 맞는 넓은 맵 구성
3. 다양한 움직임을 이용할 수 있는 스테이지 구성
4. 보스전을 위한 넓은 광장형 스테이지 구성

-전략적인 전투를 위한 4개의 다른 메커니즘의 무기 제작
1. 기본 무기 
2. 샷건 무기 여러개의 발사체가 나가는 무기
3. 라이플 무기 연사가 가능한 무기
4. 플라즈마 무기 범위형 무기

-몰입감 있는 플레이를 위한 각 행동 및 상호작용별 사운드 구현
1. 플레이어 행동별 사운드 구현
2. 적 몬스터 행동별 사운드 구현
3. 맵 오브젝트 상호작용별 사운드 구현
4. 기본 배경음악 및 전투 시 음악 구현

-유저간 경쟁요소를 위한 클리어시 클리어 정보를 DB에 저장 및 랭킹 시스템 구현
1. 클리어 화면 구현
2. 클리어 정보 전송 시 SQL query문을 통해 테이블 저장
3. 테이블 저장 정보를 이용하여 랭킹 출력
4. 각 버튼별 오름차순 내림차순 검색 기능 구현

4. 프로젝트 성과

-프로젝트 추진 내용
1. GITHUB desktop을 통한 개발자 협업 프로그램 활용
2. 팀원별 역량에 맞추어 프로젝트 기획을 구성
3. 팀원별 역량에 맞추어 역활 분배와 구현내용 세분화


5. 기대효과
유저들의 스트레스 해소
슈팅 게임 메니아들에게 컨텐츠 제공


5.데이터 베이스 구조
(데이터 흐름도)

아이디 클리어 시간 클리어 날짜 리뷰내용 -> 전송
등수를 추가 

(테이블 표 삽입) -> 데이터 분석서 참고
테이블 (등수, 아이디, 클리어 시간, 클리어 날짜, 리뷰내용)
예외처리 설명
아이디, 리뷰내용에 글자수 제한으로 예외처리 제어

6.화면 및 기능 설명
(메인화면 사진 삽입)
-메인화면

인게임 시작 버튼
게임을 시작하는 버튼

설정버튼
설정창으로 이동하는 버튼

랭킹버튼
랭킹창으로 이동하는 버튼

종료버튼
프로그램을 종료하는 버튼

-설정창
(설정창 사진 사입)
인 게임 마우스 감도를 조절하는 스크롤

소리 조절 스크롤
주오디오, 배경음악, 효과음
오디오 믹서를 이용하여 소리 관리

설정한 값들은 유저데이터로 저장되어 유지됨

-인게임
(인게임 중 화면 사진 삽입)
좌측 하단에 체력바와 쉴드바로 이루어짐
우측 하단엔 장비한 무기를 보여주고
탄약 정보 재장전 정보
획득한 무기를 인벤토리로 보여줌

-클리어 화면
ID와 리뷰를 작성할 수 있는 텍스트 필드가 있고
입력후 ID와 리뷰 추가 버튼 클릭 시 DB로 아이디, 클리어 시간, 클리어 날짜, 리뷰 내용이 보내짐
이후 메인화면으로 돌아감

7. 개선사항 및 향후계획
-개선사항 
짧은 기간안에 제작하다보니 게임의 분량과 표현할 수 있는 내용이 적어진것을 개선하고 싶어
플레이어 애니메이션 개선(그래픽)
게임 엔진속 쉐이더 랜더링 개선(그래픽)
적AI 행동양식 코드 개선
게임 스토리 개선 (컷신 연출, 등장인물 추가)

-향후계획
개선사항들 작업 후
DLC(확장팩)을 통해 더욱더 확장된 스테이지 구성
다양한 몬스터 추가
다양한 무기 추가
넷 코드를 추가하여 PVP 멀티플레이 모드 추가
